# Collaboration Rules

이 저장소의 작업은 아래 프로세스를 따른다.

1. 작업은 한 번에 하나의 목표만 처리한다.
2. 각 목표는 구현 후 커밋 전에 반드시 subagent 리뷰를 수행한다.
3. subagent 리뷰는 아래 5개 관점을 필수로 포함한다.
`성능`, `보안`, `코드 품질`, `정합성`, `버그 여부`
필요하면 관점을 추가할 수 있다.
4. 리뷰 결과는 상호 대조한다.
관점 간 충돌/공백이 있으면 관련 관점만 추가 재리뷰한다.
5. 재리뷰는 최대 2회까지 수행한다.
그래도 결론이 갈리면 담당자가 근거를 기록하고 타이브레이크 결정을 내린다.
6. 심각도 기준:
`Critical/High`는 커밋 전 반드시 수정한다.
`Medium`은 수정하거나, 미수정 사유/영향/추적 항목을 커밋 메시지 또는 PR 설명에 남긴다.
`Low`는 필요 시 후속 처리한다.
7. 각 목표는 별도 커밋으로 분리한다.
8. 커밋은 원칙적으로 해당 목표 변경만 포함한다.
단, `Critical/High` 수정을 위해 연관 파일 변경이 필수면 함께 포함하고 이유를 기록한다.
9. subagent 도구가 일시적으로 불가하면 동일 5관점을 수동 리뷰로 대체하고 그 사실을 기록한다.
